# CODEX TASK TEMPLATE

Este é o template oficial para todas as tarefas do Codex.

Cada sprint deve reutilizar este modelo, alterando apenas:
- Objetivo
- Escopo
- Entregas
- Critérios de aceite
- Validações específicas

## Estrutura obrigatória
1. Documentos de referência
2. Objetivo
3. Prioridade máxima
4. Fora de escopo
5. Entregas
6. Restrições arquiteturais
7. Critérios de aceite
8. Testes
9. Validação
10. Quality Gate
11. Relatório final

Princípios:
- menor alteração possível;
- reutilizar código;
- sem overengineering;
- sem refatorações desnecessárias;
- arquitetura guiada por docs/NORTH_STAR.md e docs/MASTER_ARCHITECTURE.md;
- uma task por prompt;
- parar ao concluir a task.
